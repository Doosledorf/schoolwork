Regular Expressions

Meta-characters

The intermediate tutorial covered the construction of pattern strings using literalcharacters. Here, you will be using meta-characters - characters, preceded with a backslash, that have special meanings inside patterns.

Below are of some of the most frequently used meta-characters in VIM:

\s whitespace character \S non-whitespace character
\d digit \D non-digit
\w word character \W non-word character
\a alphabetic character \A non-alphabetic character
\u uppercase character \U non-uppercase character

For example, to match a date such as 12/2/2014 you can use: \d\d/\d\d/\d\d\d\d

To match a 5 letter word starting with an uppercase letter, use: \u\w\w\w\ However, if you do not know how many characters are in a word, you can use quantifiers.

Quantifiers

Quantifiers allow you to set how many times a certain part of your pattern should be repeated.

Quantifier Description
* matches 0 or more of the preceding characters, ranges or metacharacters .* matches everything including empty line
\+ matches 1 or more of the preceding characters
\= matches 0 or 1 more of the preceding characters
\{n} matches exactly n times of the preceding characters

Character ranges

Now, you will be introduced to some typical character ranges.

The expressions, [012345] and [0 â€“ 5] will match any of the numbers in the range enclosed in the square brackets.

Note that the range represents just one character in the search pattern. Therefore, [0123] and 0123 do not yield the same results.

The order is not important when using the square braces. For example, [3210] and [0123] might be ordered differently but are the same character ranges. However, 0123 and 3210 are very different.

Character ranges can be particularly useful when you want to run a grammar check. You can achieve this by: \.\s\+[a-z]

This looks for a period followed by one or more blank spaces as well as a lower case word as indicated by the [a-z] range.
