About

General Introduction

Vim is a text editor, one which is configurable and efficient. It is intended to supersede the Vi editor on UNIX systems, because it sought to provide a more complete feature set. It can be seen as its own IDE for programming; programmers often use Vim and find it useful (once they’ve learned the necessary commands for efficient use). Though, it can just be a simple text editor. There isn’t a tutorial for the use of Vim in Vim itself, but help can be found using the command :help, and through various online tutorials (like this one). So, to use efficiently and to its full capacity, Vim must be learned and practiced. Vim isn’t a word processor, like Microsoft Word or Apple’s Pages, and so isn’t a WYSIWYG (what you see is what you get) editor. Designed for use in both a command-line interface environment and a graphical user interface environment as its own standalone application, it is a free and open source software under a license that includes charityware clauses, wanting users who benefit from using Vim to donate to children in Uganda through a certain charity or foundation (http://www.vim.org/sponsor/).

  • Supported OSs: Linux, Mac (shells available for download), Windows (available for download), OS/2, Amiga
  • Who uses vim?: Programmers (familiar with keyboard shortcuts will find the commands accessible), “tech enthusiasts” (those that want to learn for the sake of learning), teachers interested in vim for workflow (speeding up note-taking, coding in certain languages)
  • File formats: Vim recognizes three file formats (unix, dos, mac) that determine line ending characters

History

VIM was initially a contraction of Vi IMitated. Written by Bram Moolenar and released in November 2, 1991 by v1.14. The most stable version is 7.4, which is the current version. Bram had used the editors he had learned on Unix. When he found that there were no great Vi’s available on his new Amiga computer, he used what was available, Stevie and built upon it. The goal was to imitate VI, but it grew far beyond that and by v1.22, he donned a new name upon his work, Vi IMproved (http://www.moolenaar.net/vimstory.pdf).
